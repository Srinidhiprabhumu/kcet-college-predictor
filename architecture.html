KCET College Predictor System Architecture

System Architecture Diagram

Prediction Algorithm

Algorithm Name

This system uses a Custom Weighted Ranking Algorithm. It's a heuristic-based model designed to score and rank colleges based on a variety of factors, providing a personalized prediction.

Scoring Formula

Score = (ClosenessScore + SeatAvailabilityScore) * LocationWeight * BranchWeight

Note: The final list is capped at the top 25 colleges.

Component Breakdown

  • Closeness Score: Measures how close the user's rank is to the college's cutoff.

    ClosenessScore = e(-|Cutoff - Rank| / 10000) * 100

  • Seat Availability Score: Derived from the seat matrix, giving a boost to colleges with more seats.

    SeatAvailabilityScore = log(AvailableSeats + 1) * 10

  • Location Weight: Applies a heavy weight if the college matches a preferred location.

    LocationWeight = 5 (if match), 0.1 (if not)

  • Branch Weight: Applies a moderate weight for matching a preferred branch.

    BranchWeight = 1.5 (if match), 1 (if not)

Vertical User Flow

1. User Input

  • Input: User enters Rank, Category, Preferred Locations (dynamically), and Branches.
  • Output: A structured data object containing the user's input.
↓

2. Data Processing & Scoring

  • Input: User data object, `kcet_data.csv`, `seat_matrix.csv`.
  • Process: The Custom Weighted Ranking Algorithm calculates a score for each college.
  • Output: A sorted list of the top 25 college objects, each with a prediction score.
↓

3. AI-Powered Suggestions

  • Input: The top-ranked colleges from the previous step and the user's profile.
  • Process: A request is sent to the Gemini Service, which calls the Google AI API.
  • Output: Natural language suggestions, insights, and counselling advice.
↓

4. Display Results

  • Input: The sorted list of predicted colleges and the AI-generated suggestions.
  • Output: The Dashboard component renders the information in tables, charts, and text for the user.
